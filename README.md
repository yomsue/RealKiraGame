1. Введение и актуальность проектаДанный курсовой проект посвящен разработке Автоматизированного Рабочего Места (АРМ) «Система учета судеб» (Digital Destiny Management), стилизованного под концепцию «Тетради Смерти». Актуальность работы заключается в создании нестандартной информационной системы с использованием архитектурного паттерна и технологии прямого доступа к реляционным базам данных. В отличие от типовых систем учета (складов, библиотек), данный проект содержит элементы игровой бизнес-логики и динамического отслеживания зависимостей между действиями пользователя и состоянием системы.2. Технический стек и обоснование выбораAvalonia UI (.NET): Современный кроссплатформенный фреймворк для построения графических интерфейсов. Использование разметки XAML позволяет полностью отделить визуальный дизайн от управляющего C#-кода.СУБД SQLite: Компактная встраиваемая база данных. Файл БД (deathnote.db) хранится непосредственно в директории приложения, что обеспечивает абсолютную автономность и портативность ПО.ADO.NET: Технология доступа к данным посредством прямых SQL-запросов (SqliteCommand) и параметризированных передач данных. Это гарантирует максимальное быстродействие приложения и защиту от уязвимостей типа «SQL-инъекция».3. Структура базы данных (Реляционная модель)База данных состоит из следующих таблиц:Users: Учетные записи операторов (Богов Смерти), их пароли и баланс жизненной силы.Humans: Главный реестр жителей Земли (ID, ФИО, Возраст, Профессия, Текущий статус, Уровень греховности).DeathNoteEntries: Лог-журнал внесения записей (связывает оператора, жертву, причину смерти, текстовое описание обстоятельств и точное время).Investigations: Таблица мониторинга уровня угрозы со стороны следственного комитета L.4. Описание ключевой бизнес-логики программыМодуль интеллектуального старта: При запуске программа проверяет диск. Если база данных отсутствует, срабатывает фабричный метод генерации: кодом создаются таблицы и генерируется массив из более чем 200 уникальных жителей с разделением на мирных граждан, преступников и секретных агентов L.Модуль Глаз Бога Смерти: При вводе ФИО (или поиске по ID) программа в фоновом потоке выполняет SELECT-запрос с фильтром LIKE. Если цель жива, срабатывает алгоритм расчета оставшегося времени жизни и выгрузка морального статуса кармы (процента грехов).Система динамической угрозы (Режим L): При ликвидации обычных граждан уровень угрозы стабилен. Но при выборе сложных причин смерти (Самоубийство, Автокатастрофа) или ликвидации спецагентов из таблицы GridAgents, триггер в коде мгновенно увеличивает шкалу ProgressBar на величину до 30%. При достижении 100% срабатывает алгоритм полной блокировки интерфейса и вызов модального диалогового окна финала.
